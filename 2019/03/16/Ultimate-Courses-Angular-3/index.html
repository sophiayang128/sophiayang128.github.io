Component Routing

1. Base href and RouteModule

  1. <base href="/"> must be included in index.html to make the routing work properly
  2. RouterModule can be used to deal with all the routing, style navigation, etc.

2. Root module routes and outlet

  1. In app.module.ts, instead of imports RouterModule, we should imports its method

    • pathMatch: 'full' is used to match that empty path string
  2. router-outlet is directive via the Router which is essentially a placeholder where our components will be injected
  3. Components will be dynamically created when we change routes

3.Wildcard routes for 404 handling

  1. Any routes that do not exist on application will be using NotFoundComponent

  2. Similar to the HomeComponent:

4. Understanding routerLink

  1. routerLink is used to create navigation through pages

  2. routerLink is an Angular directive which can take in values, but also be used for data binding

5. Styling active routes

  1. routerLinkActive can add active class to the link, where we can do styling

  2. We need to use routerLinkActiveOptions to make the route to have exact match

6. Dynamic navigation with ngFor

  1. Using ngFor to dynamically create the link construct, in order to avoid specifying every link manually
